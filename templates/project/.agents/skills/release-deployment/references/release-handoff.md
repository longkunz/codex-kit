# Release Handoff

When preparing a merge, release, or deployment handoff, produce a clean summary that makes operational follow-through easy. Keep the handoff high-signal and action-oriented.

## Required Information

- **What Changed**: Summarize what changed and why. Include feature flags, env/config changes, and data migrations if relevant.
- **Validation Performed**: Confirm exactly what validation was performed and call out anything still unverified.
- **Deployment/Release State**: Detail the current rollout status and any manual release steps.
- **Operator Notes**: Capture monitoring instructions, smoke checks, or operational follow-ups. Include rollback assumptions if release risk exists.
- **Blockers/Remaining Risks**: Call out blockers or unresolved risks.
- **Follow-up Action**: Identify the exact next step needed from the operator.
